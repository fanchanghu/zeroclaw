# 智能体记忆系统深度对比：从编程专家到通用进化
**Claude Code  vs.  OpenClaw  vs.  ZeroClaw**

### 1. 核心共性：底层设计的“三大基石”
*   **Markdown 格式**：三者均支持纯文本 Markdown 存储，确保记忆的可读性、透明度与可维护性。
*   **记忆分类**：都可以分为工作记忆、场景记忆、语义记忆和程序记忆。


### 2. 核心维度对比表

| 维度 | Claude Code (编程专家) | OpenClaw (通用型) | ZeroClaw (通用型) |
| :--- | :--- | :--- | :--- |
| **设计哲学** | 领域相关、纯粹文件操作 | 领域无关、文件即记忆、混合检索 | 领域无关、分层隔离，可扩展 |
| **存储方式** | markdown 文件 | markdown 文件 | 多后端 Trait 驱动 |
| **工作记忆** | 多级压缩（细粒度、微压缩、折叠视图、自动摘要） | 支持会话裁剪、自动压缩（token数接近门限）和手动压缩（`/compact`） | 简单截断 |
| **情景记忆** | 按日期保存：`logs/2025/04/2025-04-02.md` | 按日期保存：`memory/YYYY-MM-DD.md` | 记忆类型： `Conversation`，按会话管理，支持不同后端（如SQLite） |
| **语义记忆** | 分为 `user`、`feedback`、`project`、`reference` 四类，还有索引文件：`MEMORY.md` | 直接记录在 `MEMORY.md` 文件中 | 记忆类型： `Core`，只通过工具由LLM自行创建 |
| **程序记忆** | 通过 `/skillify` 命令引导生成（仅内部开放）。 | 在用户请求时，触发自创建 Skills（工具）。 | 在会话满足特定条件（工具调用>2）时，自动触发 `SkillCreator` 流程生成。可配置 |
| **搜索技术** | 文件搜索（`GrepTool`+`GlobTool`）。 | **混合搜索**：Vector + BM25 (精准匹配) | **混合搜索**：Vector + BM25 (精准匹配) |

### 3. 关键机制与技术亮点
*   **Claude Code - “后台子代理提取”**：每个循环结束自动 fork 子代理，分析对话并静默更新记忆，不干扰用户主流程。
*   **OpenClaw - “静默记忆刷新 (Memory Flush)”**：在上下文压缩前触发特殊回合，强制模型将重要信息固化到磁盘，防止记忆丢失。
*   **ZeroClaw - “生命周期与衰减管理”**：短期记忆仅保留 7 天，长期（Core）记忆永不衰减，有效平衡信息密度与系统噪声。

### 4. 总结：应用场景建议
*   **Claude Code**：最适合**深度代码仓库管理**，其 Project 类记忆能精准捕捉架构决策。
*   **OpenClaw**：最适合**复杂知识管理**，混合搜索能在大海捞针时精准定位 ID 与特定符号。
*   **ZeroClaw**：最适合**需要能力自进化的任务**，其自动技能生成机制能实现经验到能力的闭环。
