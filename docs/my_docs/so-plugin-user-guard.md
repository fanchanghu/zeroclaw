# ZeroClaw 插件开发指南 V1---## §1 背景ZeroClaw 定义 7 类扩展点，插件实现对应 trait 即可：| trait | 用途 | runtime 已消费 ||---|---|---|| `Tool` | AI 可调用的工具 | ✅ || `Provider` | 接入 LLM 厂商 | ✅ || `Channel` | 聊天渠道 | ⚠️ 注册成功但 runtime 未查询 || `Memory` | 记忆后端 | ⚠️ || `Observer` | 观测/采集 | ⚠️ || `RuntimeAdapter` | 运行时适配 | ⚠️ || `Peripheral` | 硬件外设 | ⚠️ |> ⚠️ 标注的 5 类**当前** runtime 不会去注册表里查，所以即便插件注册成功也不会被调用。Tool 和 Provider 之外的扩展请评估再做。---## §2 选型：静态还是动态```plantuml@startumlskinparam shadowing falsestart:确定要写插件;if (能改 ZeroClaw 主仓库吗?) then (能)  :静态插件;  stopelse (不能)  :动态插件;  stopendif@enduml```| 维度 | 静态 | 动态 ||---|---|---|| 代码位置 | ZeroClaw 主仓库 | 独立项目 || 接口 | Rust trait | C ABI || 部署 | 跟随 ZeroClaw 二进制 | `.so` + `plugin.toml` || 更新 | 重编译主程序 | 替换 `.so` + 重启 |> ⚠️ **动态插件的硬约束**：必须和目标 ZeroClaw 用**完全相同的** rustc 版本、相同的 `zeroclaw-api` crate 版本、相同的全局分配器编译。Rust 类型 ABI 不稳定，跨过 C 边界的 `Box<dyn Trait>` 等载荷依赖编译期一致性。详见 [`plugin-system-handbook.md`](./plugin-system-handbook.md)。---## §3 静态插件开发静态插件 = 在主仓库新建一个 crate，写实现 + 写 register 函数 + 在 `main.rs` 调一行。### 流程```plantuml@startumltitle 静态插件开发流程（4 步）skinparam shadowing falsestart:Step 1\n创建 crate\n(cargo new + Cargo.toml + 加入 workspace);:Step 2\n实现 Tool / Provider 等 trait\n(在 src/lib.rs);:Step 3\n注册到宿主\n(src/lib.rs 写 register fn + main.rs 调用 + 主 Cargo.toml 加依赖);:Step 4\ncargo build 并验证启动日志;stop@enduml```### 步骤 1：创建 crate```bashcd /path/to/zeroclawcargo new --lib crates/zeroclaw-tool-greet```**文件：`crates/zeroclaw-tool-greet/Cargo.toml`**（替换 `cargo new` 生成的内容）```toml[package]name = "zeroclaw-tool-greet"version = "0.1.0"edition = "2024"[dependencies]zeroclaw-api = { path = "../zeroclaw-api" }async-trait = "0.1"serde_json = "1"anyhow = "1"```**文件：仓库根 `Cargo.toml`**（追加到 `[workspace] members` 数组）```toml[workspace]members = [  # ... 既有成员  "crates/zeroclaw-tool-greet",]```### 步骤 2：实现 Tool trait**文件：`crates/zeroclaw-tool-greet/src/lib.rs`**（替换 `cargo new` 生成的内容）```rustuse async_trait::async_trait;use zeroclaw_api::tool::{Tool, ToolResult};pub struct GreetTool;#[async_trait]impl Tool for GreetTool {    fn name(&self) -> &str { "my-greet" }    fn description(&self) -> &str { "Greets a person by name." }    fn parameters_schema(&self) -> serde_json::Value {        serde_json::json!({            "type": "object",            "properties": {                "name": { "type": "string" }            },            "required": ["name"]        })    }    async fn execute(&self, args: serde_json::Value) -> anyhow::Result<ToolResult> {        let name = args.get("name").and_then(|v| v.as_str()).unwrap_or("world");        Ok(ToolResult { success: true, output: format!("Hello, {name}!"), error: None })    }}```### 步骤 3：注册到宿主**文件：`crates/zeroclaw-tool-greet/src/lib.rs`**（在文件末尾追加）```rustuse zeroclaw_api::plugin::{PluginRegistry, RegistrySet};pub fn register(registries: &RegistrySet) {    registries.tools.register("my-greet", |_cfg| Ok(Box::new(GreetTool)));}```**文件：`src/main.rs`**（找到 `zeroclaw_fixture_static::register(&registries)` 附近，追加一行）```rustzeroclaw_tool_greet::register(&registries);```**文件：根 `Cargo.toml`**（在主 crate 的 `[dependencies]` 段追加）```tomlzeroclaw-tool-greet = { path = "crates/zeroclaw-tool-greet" }```### 步骤 4：编译并验证```bashcargo buildcargo run -- agent -m "greet me as Alice"```启动日志含 `static plugin registered: my-greet` 即成功。---## §4 动态插件开发动态插件 = 独立 cargo 项目编译成 `.so`，加一份 `plugin.toml` 应用清单，部署到 ZeroClaw 配置的目录。### 流程```plantuml@startumltitle 动态插件开发流程（4 步）skinparam shadowing falsestart:Step 1\n创建项目\n(cargo new + Cargo.toml 设 cdylib);:Step 2\n实现 Tool / Provider 等 trait\n(在 src/lib.rs);:Step 3\n注册到宿主\n(src/lib.rs 写工厂 + FFI 导出 + 写 plugin.toml + 部署 + 配置 native_paths);:Step 4\ncargo build --release 并验证启动日志;stop@enduml```> 与静态插件 4 步完全对应：步骤 1/2/4 概念一致，步骤 3 都是"告诉宿主在哪能找到我"——只是机制不同（静态靠 main.rs 调用，动态靠 FFI 导出 + plugin.toml）。### 步骤 1：创建项目```bashcargo new --lib my-translate-plugincd my-translate-plugin```**文件：`Cargo.toml`**（替换 `cargo new` 生成的内容）```toml[package]name = "my-translate-plugin"version = "0.1.0"edition = "2024"[lib]crate-type = ["cdylib"]   # 关键：编译成 .so / .dylib / .dll[dependencies]zeroclaw-api = { git = "https://github.com/your-org/zeroclaw", tag = "v0.7.3" }async-trait = "0.1"serde_json = "1"anyhow = "1"```### 步骤 2：实现 Tool trait**文件：`src/lib.rs`**（替换 `cargo new` 生成的内容；写法与静态插件完全相同）```rustuse async_trait::async_trait;use zeroclaw_api::tool::{Tool, ToolResult};pub struct TranslateTool;#[async_trait]impl Tool for TranslateTool {    fn name(&self) -> &str { "my-translate" }    fn description(&self) -> &str { "Translates text" }    fn parameters_schema(&self) -> serde_json::Value {        serde_json::json!({            "type": "object",            "properties": {                "text":   { "type": "string" },                "target": { "type": "string" }            },            "required": ["text", "target"]        })    }    async fn execute(&self, args: serde_json::Value) -> anyhow::Result<ToolResult> {        let text   = args["text"].as_str().unwrap_or("");        let target = args["target"].as_str().unwrap_or("en");        Ok(ToolResult {            success: true,            output: format!("[{target}] {text}"),            error: None,        })    }}```### 步骤 3：注册到宿主包含三件事：写工厂函数、写两个 FFI 导出符号、写 `plugin.toml`、配置 `native_paths`。#### 3.1 工厂函数 + FFI 导出**文件：`src/lib.rs`**（在 trait 实现之后追加）```rustuse core::ffi::c_void;use zeroclaw_api::plugin::PluginHandle;use zeroclaw_api::version::API_VERSION_U32;// ─── 工厂函数（双 Box 模板，照搬即可） ─────────────────────unsafe extern "C" fn translate_factory(    _config_json: *const u8,    _config_len: usize,    out_tool: *mut *mut c_void,) -> i32 {    if out_tool.is_null() { return 1; }    let inner: Box<dyn Tool> = Box::new(TranslateTool);    let outer: Box<Box<dyn Tool>> = Box::new(inner);    unsafe { *out_tool = Box::into_raw(outer) as *mut c_void };    0   // 0 = 成功；任何非零都视为失败}// ─── 必须导出的两个 FFI 符号 ──────────────────────────────#[unsafe(no_mangle)]pub extern "C" fn zc_api_version() -> u32 {    API_VERSION_U32}#[unsafe(no_mangle)]pub unsafe extern "C" fn zc_register_plugins(handle: *mut PluginHandle) {    if handle.is_null() { return; }    let h = unsafe { &*handle };    let name = b"my-translate";    unsafe {        (h.register_tool)(h.inner, name.as_ptr(), name.len(), translate_factory);    }}```> 关于"双 Box"为什么这样写——是为了把 16 字节的 `Box<dyn Tool>` 胖指针穿过 C ABI 边界。机制细节见 [`plugin-system-handbook.md`](./plugin-system-handbook.md) §2.4。**作为插件作者照搬模板即可**。#### 3.2 plugin.toml**文件：`plugin.toml`**（与 `Cargo.toml` 同级，部署时与 `.so` 一起放到目标机器）```tomlname = "my-translate-plugin"        # 插件唯一名（建议加唯一前缀防撞，例如 acme-translate）version = "0.1.0"                    # 你的插件版本，自己定api_version = 100                    # 来自 zeroclaw_api::version::API_VERSION_U32description = "Translate text"       # 可选library = "./libmy_translate_plugin.so"   # 可选，不写则按 lib<name>.so 自动查找```| 字段 | 必填 | 由谁定 ||---|---|---|| `name` | ✅ | 你 || `version` | ✅ | 你（你的插件版本） || `api_version` | ✅ | ZeroClaw 维护者（来自 `API_VERSION_U32`） || `description` / `library` | ❌ | 你 |#### 3.3 部署 + 配置 `native_paths`把 `plugin.toml` 和编译产物 `.so` **放在同一个目录**：```/opt/zeroclaw-plugins/my-translate/├── plugin.toml└── libmy_translate_plugin.so```**文件：用户的 ZeroClaw 主配置 `config.toml`**（不是你的 plugin.toml）```toml[plugins]native_paths = ["/opt/zeroclaw-plugins/my-translate"]```### 步骤 4：编译并验证```bashcargo build --release# 产物路径：#   Linux:   target/release/libmy_translate_plugin.so#   macOS:   target/release/libmy_translate_plugin.dylib#   Windows: target/release/my_translate_plugin.dllzeroclaw agent -m "translate hello to japanese"```启动日志含 `loaded native plugin: my-translate-plugin` 即成功。失败时按 [§7 错误排查](#§7-错误排查) 定位。---## §5 多组件 / 跨类型插件一个插件同时注册多个 Tool / 跨类型注册 Tool + Provider 等，结构和单组件一样，只是注册时多调几次：### 静态版**文件：`crates/your-plugin/src/lib.rs`** 的 `register` 函数：```rustpub fn register(registries: &RegistrySet) {    registries.tools.register("my-tool-a", |_| Ok(Box::new(ToolA)));    registries.tools.register("my-tool-b", |_| Ok(Box::new(ToolB)));    registries.providers.register("my-llm", |_| Ok(Box::new(MyProvider)));}```### 动态版**文件：`src/lib.rs`** 的 `zc_register_plugins`：```rust#[unsafe(no_mangle)]pub unsafe extern "C" fn zc_register_plugins(handle: *mut PluginHandle) {    if handle.is_null() { return; }    let h = unsafe { &*handle };    let n1 = b"my-tool-a";    unsafe { (h.register_tool)(h.inner, n1.as_ptr(), n1.len(), tool_a_factory); }    let n2 = b"my-tool-b";    unsafe { (h.register_tool)(h.inner, n2.as_ptr(), n2.len(), tool_b_factory); }    let n3 = b"my-llm";    unsafe { (h.register_provider)(h.inner, n3.as_ptr(), n3.len(), llm_factory); }}```每个 `xxx_factory` 都按 §4 步骤 3.1 的双 Box 模板复制一份，把 `Box::new(TranslateTool)` 换成对应的实现即可。---## §6 配置处理（**重要约束**）### 现状ZeroClaw runtime 调用工厂时**总是传 `Value::Null`**（来源：[`crates/zeroclaw-providers/src/lib.rs:1764-1766`](../AIOS/zeroclaw/crates/zeroclaw-providers/src/lib.rs#L1764)）。也就是说，**目前你的工厂函数收到的 `_config_json` 永远是 4 字节 `"null"`**——通过 `plugin.toml` 或 `config.toml` 给插件传定制配置的链路尚未接通。### 计划中的接通方式未来 runtime 改动后，将从用户 `config.toml` 中按插件名提取该插件的配置子表，序列化为 JSON 后传给工厂。设想中的用户配置形态：```toml[plugins.providers.my-translate]api_key = "AIza..."endpoint = "https://my-gateway.example.com"timeout_secs = 30```宿主会把 `[plugins.providers.my-translate]` 这一段序列化为 JSON 后传给 `translate_factory`。### 给开发者的建议- **不要自己设计 workaround**（如读环境变量、读独立配置文件）——这些会在官方接通后变成需要清理的临时代码。- 如果你的插件**当前必须**有配置才能工作（如需 API key），等官方接通后再开发，或者评估接受"工厂阶段拿 Null、运行阶段失败"的现状。- 关注 ZeroClaw 主仓库 issues 里的"plugin config wiring"相关进展。未来接通后**业务代码改动极小**：把 `_config_json` 前的下划线去掉，用 `serde_json::from_slice` 反序列化为你定义的 struct，trait 实现完全不用动。---## §7 错误排查启动日志里看到 `failed to load plugin: ZeroClawError::*` 时按下表定位：| 错误 | 关键字 | 含义 | 修复 ||---|---|---|---|| `VersionMismatch` | "major version mismatch" | manifest 或 .so 内 `api_version` 与宿主主版本号不一致 | 改 toml 的 `api_version`；或重新编译插件用最新 `zeroclaw-api` || `VersionMismatch` | "newer than host" | 插件 minor > 宿主 minor | 升级 ZeroClaw；或降级插件依赖 || `DynLibLoad` | "failed to load dynamic library" | `.so` 文件不存在/损坏，或缺系统库 | `ldd libxxx.so` 看缺什么 || `DynLibLoad` | "failed to read/parse plugin manifest" | toml 路径或语法错 | 检查路径权限，`taplo check` 校验 || `MissingSymbol` | "Missing symbol: zc_api_version" | 缺版本符号导出 | 加 `#[unsafe(no_mangle)] pub extern "C" fn zc_api_version` || `MissingSymbol` | "Missing symbol: zc_register_plugins" | 缺注册符号导出 | 同上模式 || `ComponentNotFound` | "not found. Available: ..." | runtime 查询的名字未注册 | 检查工具名拼写；确认 `zc_register_plugins` 真的被调用 || `ComponentInit` | "rc=N" | 工厂函数返回非零 | 检查工厂内部逻辑 |调试日志：```bashRUST_LOG=zeroclaw_loader=trace,zeroclaw_runtime=debug zeroclaw agent```跨平台找 dlopen 失败原因：```bashLD_DEBUG=libs zeroclaw agent 2>&1 | grep my_plugin       # LinuxDYLD_PRINT_LIBRARIES=1 zeroclaw agent 2>&1 | grep my_plugin   # macOS```---## §8 速查### 静态插件 4 步1. `cargo new --lib crates/your-plugin`，加入 workspace2. 实现 trait（`src/lib.rs`）3. 写 `pub fn register(...)` + 主程序 `main.rs` 调一行 + 主 `Cargo.toml` 加依赖4. `cargo build` 并验证日志### 动态插件 4 步1. `cargo new --lib your-plugin`，`Cargo.toml` 设 `crate-type = ["cdylib"]`2. 实现 trait（`src/lib.rs`）3. 写工厂函数 + `zc_api_version` + `zc_register_plugins` + `plugin.toml` + 部署 + 配置 `native_paths`4. `cargo build --release` 并验证日志### 7 类组件入口| trait | 静态 register | 动态回调字段 ||---|---|---|| `Tool` | `registries.tools.register` | `h.register_tool` || `Provider` | `registries.providers.register` | `h.register_provider` || `Channel` | `registries.channels.register` | `h.register_channel` || `Memory` | `registries.memory.register` | `h.register_memory` || `Observer` | `registries.observers.register` | `h.register_observer` || `RuntimeAdapter` | `registries.runtimes.register` | `h.register_runtime` || `Peripheral` | `registries.peripherals.register` | `h.register_peripheral` |### 仓库内参考实现- `crates/zeroclaw-fixture-static/src/lib.rs` — 静态插件最小实现- `crates/zeroclaw-fixture-dynamic/src/lib.rs` — 动态插件最小实现---## 附录 A：可复制模板### A.1 静态插件最小骨架**文件：`crates/your-plugin/src/lib.rs`**```rustuse async_trait::async_trait;use zeroclaw_api::plugin::{PluginRegistry, RegistrySet};use zeroclaw_api::tool::{Tool, ToolResult};pub struct MyTool;#[async_trait]impl Tool for MyTool {    fn name(&self) -> &str { "my-tool" }    fn description(&self) -> &str { "..." }    fn parameters_schema(&self) -> serde_json::Value {        serde_json::json!({ "type": "object" })    }    async fn execute(&self, args: serde_json::Value) -> anyhow::Result<ToolResult> {        Ok(ToolResult { success: true, output: args.to_string(), error: None })    }}pub fn register(registries: &RegistrySet) {    registries.tools.register("my-tool", |_cfg| Ok(Box::new(MyTool)));}```### A.2 动态插件最小骨架（单 Tool）**文件：`src/lib.rs`**```rustuse core::ffi::c_void;use async_trait::async_trait;use zeroclaw_api::plugin::PluginHandle;use zeroclaw_api::tool::{Tool, ToolResult};use zeroclaw_api::version::API_VERSION_U32;pub struct MyTool;#[async_trait]impl Tool for MyTool {    fn name(&self) -> &str { "my-tool" }    fn description(&self) -> &str { "..." }    fn parameters_schema(&self) -> serde_json::Value {        serde_json::json!({ "type": "object", "additionalProperties": true })    }    async fn execute(&self, args: serde_json::Value) -> anyhow::Result<ToolResult> {        Ok(ToolResult { success: true, output: args.to_string(), error: None })    }}unsafe extern "C" fn my_tool_factory(    _cfg: *const u8, _len: usize, out: *mut *mut c_void,) -> i32 {    if out.is_null() { return 1; }    let inner: Box<dyn Tool> = Box::new(MyTool);    let outer: Box<Box<dyn Tool>> = Box::new(inner);    unsafe { *out = Box::into_raw(outer) as *mut c_void };    0}#[unsafe(no_mangle)]pub extern "C" fn zc_api_version() -> u32 { API_VERSION_U32 }#[unsafe(no_mangle)]pub unsafe extern "C" fn zc_register_plugins(handle: *mut PluginHandle) {    if handle.is_null() { return; }    let h = unsafe { &*handle };    let name = b"my-tool";    unsafe { (h.register_tool)(h.inner, name.as_ptr(), name.len(), my_tool_factory); }}```### A.3 plugin.toml 模板**文件：`plugin.toml`**```tomlname = "my-plugin"version = "0.1.0"api_version = 100description = "Optional one-line
